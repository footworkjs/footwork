Welcome to footwork.js!

footwork.js is a front-end javascript framework based off of several open source libraries, most notably including knockout.js, postal.js, lodash, riveter, and requirejs.

It can be thought of (feature wise) to be in the same realm as Angular, Ember, Marionnette, Durandal, and others. It is meant to provide a basis (the footing, if you will) of a large/ambitious web application, based on a combination of several well known open source libraries and design patterns.

A broad overview of features included with footwork:

  • Custom components.
    • Automatic lazy loading of components.
    • Registered pre-caching of components.
    • Lifecycle management.
    • Integrated namespace based CQRS.
    • Retains support for native/standard knockout component registration.
  • Formalized viewModel concept with automatic scope wrapping.
    • Automatic lazy loading of viewModels.
    • Registered pre-caching of viewModels.
    • Lifecycle management.
    • Integrated namespace based CQRS.
    • Context binding management via embedded custom <viewModel /> component or direct applyBindings() calls.
  • Namespace based CQRS support on a per-viewModel, per-component, or custom basis.
    • Lifecycle management for all handlers.
    • Configurable automatic namespace assignment for newly created viewModels and components.
    • Support for commands, requests, events, and pub/sub.
    • Supports 'durable' (long living) commands and events based on either a time based expiration, raw count, or a permanent basis.
  • Routing, including nested routing support.
    • Infinitely nested and context aware router/child-router support.
    • Custom named outlet support (analogous to emberjs outlet support).
    • Named parameters in route definitions.
  • inter-viewModel / inter-component dependency tracking and communication.
    • Based on integrated namespace support.
    • Track dependencies, share state and information between viewModels and components cleanly.
    • Configurable as read-only or writable shared data.
  • Behavioral mixins, and inheritance.
    • Reusable behaviors and logic expressed through an easy to use mixin syntax.
    • viewModel inheritance.
