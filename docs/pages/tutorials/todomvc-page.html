Introduction

Getting started with footwork is best done with a real application. In this tutorial we will be covering the creation of a TODO list app.

Specifically, in relation to footwork.js we will cover the following topics:

  • Downloading dependencies and setting up an environment to use footwork.
  • App instantiation.
  • RequireJS usage for automatic dependency management.
  • <viewModel> wrapping of HTML and 2-way binding.
  • <custom-component> creation and usage.
  • Broadcastable and receivable usage.
  • Routing and state initialization.

Getting Started

This tutorial has a companion repository on github which houses the source. It is recommended (although not 100% necessary) that you follow along and play with the code as you progress through the tutorial.

This section will not cover setting up a webservice or basic git usage (beyond the few commands needed to use the tutorial). If you are new to git, I would recommend taking a look at the walk-through available on github.

You can clone the tutorial repo with the following command:

git clone https://github.com/reflectiveSingleton/todo-tutorial.git

This repo has all of the assets needed to run the application. You may notice it also makes use of bower bower for package management, and less.js for CSS. It is not necessary to know anything about these in order to use this tutorial however they are both great tools worthy of checking out.

Prerequisites

This tutorial assumes a basic understanding of the following topics:

  • HTML
  • CSS
  • Javascript

Although not required, it is helpful if you have knowledge of any of the dependencies footwork uses, most notably knockout. You shouldn't have too much issue getting through the tutorial without it, but having an idea of how to use observables and 2-way data binding at the least would be helpful.

This tutorial will not cover the creation of the CSS/LESS, they are included for you in the base repository. Also of note, the javascript runtime of LessCSS is used so that it needs no re-compile step if/when you alter it.

Setup

After cloning the repository you are ready to get started, begin by checking out the setup step branch:

git checkout -f setup

This will set your workspace up with the base application structure and code. Open up the page in your browser and you won't see anything too interesting just yet...but we'll get to that.
